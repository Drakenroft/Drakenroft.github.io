Il Network Engine per tutte le maggiori piattaforme

Photon incorpora completamente il complesso strato di networking di ogni piattaforma client per te. Photon Server supporta UDP, TCP, HTTP e Web Socket affidabili con un protocollo di trasferimento leggero e snello per ciascuno di essi.I client di gioco comunicano tra piattaforme e protocolli: basta inviare i dati e Photon esegue la deserializzazione / serializzazione, non tu. Photon Server Architecture

Il Photon Core è scritto in C ++ nativo per fornire le massime prestazioni e consentire il supporto di toolchain di prima classe. In combinazione con le porte di completamento di Input/Output di Windows Server (IOCP) implementa il pesante "sollevamento" dei protocolli. La "business logic" viene eseguita in un CLR .NET e contiene le applicazioni scritte in C# o in qualsiasi altro linguaggio .NET.
Tutte le applicazioni si trovano in cima a un framework RPC espandibile e a degli eventi che risolvono attività comuni. Le chiamate RPC sono semplici e flessibili mappate alle istanze di operazione. La filettatura è semplificata, in base ai principi di trasmissione dei messaggi che utilizzano le fibre. Nel complesso, Photon Server è progettato per saturare il traffico prima che la CPU inizi a fare da "collo di bottiglia".



Scalabilità

Intesa come la possibilità/necessità di scalare facilmente l'applicazione su più server man mano che i titoli crescono.
Photon Server contiene un bilanciamento del carico (Master / Lobby) per consentire un'elevata scalabilità. Il server Master / Lobby tiene traccia del livello di carico sui Game Server e dei giochi attualmente aperti e indirizza i giocatori di conseguenza per un QoS (Quality of Service)) ottimale. I Game Server ospitano le partite e segnalano regolarmente il carico di lavoro corrente e l'elenco delle sessioni di gioco in esecuzione sul server principale. I giochi basati su stanze sono immediatamente disponibili con l'applicazione Load Balancing (bilancio del carico) fornita nel codice sorgente. Gli MMO (Massive Multiplayer Online) di solito richiedono un concetto di ridimensionamento personalizzato che soddisfi le esigenze individuali. Trova un framework MMO che includa la gestione degli interessi è fornito con l'SDK di Photon Server.

Hosting and Operations

Combina server self hosted e cloud per fornire capacità sufficienti man mano che i tuoi titoli crescono. Esegui Photon Server su server bare metal, server cloud o persino mashup sia per un ottimo rapporto prezzo / prestazioni. Ad esempio, coprire carichi elevati aggiungendo al volo alcuni server cloud alle macchine dedicate. Photon Server si integra facilmente con il tuo panorama di Windows Server. Eseguirlo come servizio Windows, copiarlo usando PowerShell e controllare i contatori delle prestazioni di Windows inclusi per monitorare il carico del server.

Offuscazione del codice sorgente

L'offuscazione del codice sorgente, è una procedura che viene eseguita per aumentare la sicurezza del proprio codice, sicurezza intesa come una difficoltà maggiore da parte di terze entità nel cercare di capire, clonare e rivendere il proprio codice in maniera illegale e non autorizzata.

Offuscare il codice rende inoltre più difficoltoso il "Reverse Engineering", il quale permette sempre a terze entità di replicare il codice in maniera legale.

In molti casi l'offuscazione del codice non ha soltanto una funzione di protezione del codice ma anche di aumento delle performance, è stato evidenziato infatti da molti benchmark che un offuscatore avanzato (Ex ProGaurd per il Java),



Un Offuscatore molto utilizzato con Unity

può aumentare le performance fino anche del 20% in quanto il bytecode generato è più snello e facile da leggere per la virtual machine, inoltre a seconda dell'offuscatore può modificare parte del nostro codice con versioni che lui ritiene più efficienti (Ex utilizza le Lambda ove non le abbiamo utilizzate), quest'ultima può essere disattivata. Tra le funzioni principali di un qualsiasi Offuscatore troviamo:
•Rinominare variabili, classi e metodi, con nomi incomprensibili.
•Rimuovere tutti i commenti.
•Criptazione delle stringe (String Obfuscation), presente però solo in alcuni e con algoritmi specifici.
Gli Offuscatori sono software utilizzanti prevalentemente con linguaggi di alto livello che generano codice intermediario/bytecode per i quali siano usciti decompilatori, che possono mettere a rempetaglio la sicurezza del nostro software. Nel caso di Unity che utilizza il C# se lasciamo il nostro codice IL scoperto, chiunque può risalire al nostro codice C# e modificare e ricompilare il codice, creando hack e vantaggi non previsti da noi per il nostro gioco/software, una cosa che si vuole assolutamente prevenire.

Sistemi di Anti-Cheat

I sistemi di Anti-Cheat sono framework i quali si occupano di prevenire l'hacking/cracking ed il cheating nei propri applicativi online. E' molto importante proteggere i propri applicativi con sistemi di anti-cheat idonei in quanto, presenza dei sistemi di hack come i più comuni aimbot/wallhack hanno un alta probabilità di manifestarsi nel corso della vita dell'applicativo sopratutto se esso acquisice una certa notorità (ma anche quando non la raggiunge, spesso chi usa mezzi di cheating naviga in giochi/applicativi poco conosciuti appunto per evitare ban su controparti più famose).

C'è da aggiungere che i sistemi di Anti-Cheat non riguardano solamente la protezione da hacker in giochi online, ma anche il cercare di combattere la pirateria, in quanto esistono sistemi come i DRM (Digital Rights Management) che si occupano di rendere il software molto più difficile da crackare/piratare tramite frequenti check online della validità della licenza, solitamente i DRM comunicano con piattaforme molto famose e stabili (Ex Steam/Uplay/Origin).

Un Anti-Cheat in passato molto famoso

E' quindi necessario prendere delle idonee contromisure, a tal proposito i sistemi di Anti-Cheat forniscono un ottimo lavoro, le principali funzioni che uno qualsiasi di essi integra sono:
•Protezione delle variabili in memoria.
•Riconoscimento e compromissione dei sistemi di cheating più famosi come speedhack/aimbot/wallhack/time cheating tramite delle informazioni di output di "false positive", le quali fanno credere al cheat di essere in esecuzione correttamente anche quando non è così.
•Esplusione dal server e eventuale ban/sospensione del cheater dall'applicativo (il tipo di punizione può essere configurata anche a seconda del numero e di quale cheat il cheater ha fatto utilizzo).
•Offuscazione del codice (simil offuscatore).
•E alcuni a seconda del framework scelto possono includere offuscatori completi, per rendere più difficile decifrare il codice, e creare hack per quello specifico gioco/applicativo.
